# Branch Management

**Create feature branch BEFORE any file changes.** Check with `git branch --show-current`.

**Naming:** `{type}/{work-item-id}-{short-description}`

**Types:** `bugfix/`, `feature/`, `hotfix/`, `docs/`, `chore/`

**Pending:** `{type}/pending-{desc}` â†’ rename after work item creation.

**Branch Policies (recommended):**
- Require PR for merges
- Require minimum reviewers
- Require successful build

**Rules:** One branch per work item.
